Title: Endpoint to check container status returns generic ERROR status

=== CONTENT ===

Grant (Original Poster):
We are running into an issue where we get back generic "ERROR" code when attempting to check the status of a video container we just uploaded. This is using Instagram API via Instagram Login.

We are uploading videos to Instagram per the documentation, but we sometimes get this result when checking the container status:

Request:
GET https://graph.instagram.com/v22.0/<container_id>?fields=id,status,status_code&access_token=<token>

Response:
{'id': '18482264659064819', 'status': 'ERROR', 'status_code': 'ERROR'}

According to the documentation, if the status_code is ERROR then this endpoint is supposed to give us back a status with detail about why the container failed to upload.

I can't find any pattern to when this issue happens. It doesn't seem to be a resolution or encoding issue, because we encode all videos as 1080p mp4s before uploading.

This is how we are publishing the media:
POST https://graph.instagram.com/v22.0/<id>/media?caption=hello&is_carousel_item=False&media_type=REELS&video_url=<url>&share_to_feed=True&access_token=<token>

Does anyone know why this might be happening?
