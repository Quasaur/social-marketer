Bas (Selected Answer - Upvote 16):
The real problem is different and not on the /media_publish API.
Error code 9007 is correct.
It's not possible to publish container with status code IN_PROGRESS (In progress status: the media is still being processed).

The real problem that started on September 6th is that there's an issue on the Meta side related to reel contaners processing. The containers remain in IN PROGRESS status without ever transitioning to FINISHED, and after 24 hours, they transition to EXPIRED.

Check your container status with /[container_id]?fields=id,status_code,status and you can see the IN PROGRESS status.

I can confirm that we are also experiencing this issue (containers stuck in the IN PROGRESS state), which is preventing a large number of reels from being published.

Same problem was happened on may 2025.

Joseph (Detailed Report - Dec 12, 2025):
The API returns error code 9007 (Media ID is not available) even though the media is eventually processed successfully on Instagram's side. Retrying the exact same request after a short delay consistently succeeds without any changes.

This behavior strongly suggests a timing or media processing issue within the Instagram API rather than an invalid request.

Error Details:
{
  "error": {
    "message": "Media ID is not available",
    "type": "OAuthException",
    "code": 9007,
    "error_subcode": 2207027,
    "is_transient": false,
    "error_user_title": "Cannot Publish",
    "error_user_msg": "The media is not ready for publishing, please wait for a moment"
  }
}

Observed Behavior:
- The media upload request is accepted.
- The publish request fails with error 9007.
- No changes are made to the request or payload.
- Retrying the same request manually after 1-5 times succeeds.

This occurs intermittently and is reproducible across multiple users and accounts. The issue affects images (not only videos).

Expected Behavior:
If media is still processing on Instagram's side:
- The API should either delay publication internally, or
- Return a transient error with clear retry guidance.

Alternatively, the publish endpoint should block until media is ready instead of returning a hard error.

Impact:
- End users perceive this as a publishing failure and downgrade product ratings.
- Automation workflows fail despite eventually succeeding on retry.
- Third-party platforms are forced to implement arbitrary retry logic to work around API timing issues.
